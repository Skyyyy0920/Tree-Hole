# 一、基本信息

1. 标题：*Kamino: Constraint-Aware Differentially Private Data Synthesis*

2. 发表时间：2021

3. 出版源：VLDB

4. 领域：Private Data Cleaning

5. 摘要：

    企业越来越多地依赖数据来支持决策。

    当数据包含隐私和敏感信息时，数据所有者通常希望发布与真实数据同样有用的合成数据库实例，同时确保个别数据记录的私密性。现有的差异私有数据合成方法旨在基于应用程序生成有用的数据，但它们未能保持结构化数据最基本的数据属性之一——元组和属性之间的潜在相关性和依赖性(即数据的结构)。这种结构通常表示为完整性和模式约束，或者表示为概率生成过程。因此，合成的数据对于任何需要保留该结构的下游任务都没有用处。

    这项工作提出了Kamino，一个数据合成系统，以确保不同的隐私，并保留原始数据集中的结构和相关性。Kamino将数据库实例及其模式(包括完整性约束)作为输入，并生成具有不同隐私性和结构保存保证的合成数据库实例。我们的经验表明，Kamino在保留数据结构的同时，在训练分类模型和回答边缘查询的应用中，比最先进的差分私有数据合成方法获得了相当的甚至更好的有用性。

# 二、研究背景

## 1. 问题定义

从音乐推荐到冠状病毒研究，各种组织一直广泛依赖个人数据来支持越来越多的业务。这类数据通常是结构化的，包含个人的敏感信息。对于数据驱动的应用和研究，不计后果地共享数据会导致极大的隐私担忧和处罚。差异隐私 (Differential
privacy, DP) 已成为政府机构和企业的标准数据隐私保障。不正式地说，无论是否使用个人的数据进行计算，满足DP的数据共享过程的输出都具有类似的分布。因此，输出不能用来推断任何个人的数据，因此被认为是“私有的”。

差别隐私通常通过随机化来实现，例如根据所需的隐私级别向输入数据注入受控噪声，因此在隐私和该数据对下游应用程序的效用之间存在权衡。在之前的工作中，通常采用的一种方法是针对给定的应用程序(例如，发布统计数据，构建预测模型，回答SQL查询)优化这种权衡。例如，APEx是为数据探索而设计的;对于每个查询，APEx搜索能够以最小的隐私代价准确回答查询的最佳差分私有算法。这种工作允许对算法进行微调，以在隐私成本和给定应用程序的准确性之间实现最佳权衡，但发布的输出可能对其他应用程序没有用处。在同一数据集上运行新应用程序通常需要额外的隐私成本。

一种有吸引力的替代方法是发布一个差异私有的合成数据库实例，该实例具有一组所需的属性，比如类似的值分布或依赖结构，希望它对需要这些属性的大量下游应用程序具有相同的效用或与原始数据集一样有用。例如，美国人口普查局(US Census Bureau)发布了不同的私人人口普查数据，它已被证明，保持类似的家庭-工作场所分布作为填充地图应用程序的真实数据是有用的。私下发布合成数据避免了为每个目标应用程序设计单独的机制，并且由于差异隐私的后处理特性，所有支持的应用程序只产生一次隐私成本。

## 2. 主要贡献

- 我们相信这是第一个考虑差异私有数据合成中的拒绝约束的工作，这是结构化数据的重要属性。我们使用概率数据库框架来合并数据中心和属性相关性。

- 我们根据给定的约束(§3和§4)，通过将概率数据库模型分解为一系列子模型，为Kamino开发了一种高效的学习和采样算法。

- 我们在Kamino中设计了一个私有学习算法来学习给定dc的权重，从而允许在模型中将它们解释为软约束(§5)。

- 我们为Kamino构建了原型，这是第一个端到端用数据中心生成差异隐私数据的系统，并应用先进的隐私组合技术来获得严格的端到端隐私约束(§6)。

- 我们在现实世界的数据集上评估Kamino数据，并表明合成数据与真实数据中给定的dc有类似的违反，并且与最先进的方法(§7)相比，它们在边缘查询(变化距离)和学习任务(准确性和F1)中也实现了最佳或接近最佳的数据有用性。

## 3. 相关工作

    关于发布差异私有合成数据已有大量文献[17,36,61,84]。这些方法可以分为两类:1)统计方法，专注于合成低维投影;2)深度学习方法，训练一个深度生成模型对元组进行采样。

    这两个类都假定元组是id，因此不能保留数据的结构。我们的方法是两者的结合，更重要的是，我们的方法与之前的工作不同，因为我们显式地考虑了元组之间强制的拒绝约束[48]，而不是简单地假设元组独立。

    生成合成数据的统计方法通常估计低维边际分布[66,80]，这是由于私有化具有不同隐私保障的高维数据的难度[16,31,39,75]。基于属性间条件独立性的假设，这些低维分布可用于估计高维元组分布，可使用概率图形模型[52]建模，如使用贝叶斯网络[56,65,83]或无向图[21,57]。在该模型下，只可能捕获依赖属性之间的相关性，而在之前的工作中，没有捕捉到广泛存在于条件独立属性和元组之间的相关性。

    深度学习模型已被广泛应用于合成非结构化数据，如图像[70]、视频[19]和自然语言[44]。与非结构化数据不同，结构化数据是使用关系模式定义的，因此，结构相关性自然存在。Naïvly在结构化数据上应用深度学习模型，如GAN[42]和自动编码器[51]，至少面临两个挑战。首先，这些模型通常以数值向量作为输入，而流行的编码方案如单热编码或顺序编码对结构化数据[35]都不能很好地工作。其次，与统计方法类似，基于深度模型的方法(如[38,50,73,81])存在结构相关性缺失的问题。

    一般来说，由于准确性和隐私性之间的权衡，生成差异私有合成数据是困难的[16,31,39,75]。

    另一方面，高效的私有数据生成算法无法为所有查询提供相同级别的准确性保证。

    因此，现有的实用方法(例如[12,20,21,50,83])选择只私下学习关联的子集来对真实数据建模。然而，这些方法没有显式地捕获数据的结构，因此在输出中保存得很差。

# 六、总结

在这项工作中，我们的动机是设计一个合成数据生成器，可以保存数据的结构和个人数据记录的隐私。我们提出了Kamino，一个约束感知差分私有数据综合的端到端数据综合系统。Kamino将一个数据库实例及其模式(包括拒绝约束)作为输入，并生成一个合成数据库实例。实验结果表明，与最先进的方法相比，Kamino方法可以保持数据的结构，同时生成有用的合成数据，用于训练分类模型的应用和回答边缘查询。
